# Meeting Summary

## I\. Meeting Opening and Core Background

This meeting focused on the web project collaborated by six teams\. The core background is that the project has been delayed by two weeks\. The key points of the meeting were to clarify division of labor, sort out requirements, and ensure the project progresses as scheduled\. The meeting objectives were to sort out the requirement list, clarify the responsibilities of each team, clarify system interaction and data flow, and solve the core problems in the current project advancement\.

## II\. Core Medical Requirements 

The core of the project revolves around the collection and application of knee joint angle data to meet doctors needs for patients rehabilitation data\. The specific requirements are as follows:

- Necessity of measuring knee joint angles: Provide data support for patients rehabilitation and assist doctors in formulating/adjusting rehabilitation plans\.

- Definition and standards of two key angles (for healthy males):
        

    - Angle 1 (leg straight): Standard is 0 degrees;

    - Angle 2 (knee bent): Standard is 130 to 150 degrees\.

- Data source: Provided by the Sensor Team or the Data Processing Team\.

## III\. Core Responsibilities and Requirements of Each Team

### \(I\) AI Team

- Core task: Automatically recommend rehabilitation templates based on knee joint angle data (doctors will provide about 20 basic templates);

- Interface collaboration: Receive motion data from the Data Processing Team, output recommended templates through the AI interface, and clarify interface specifications with the Data Processing Team;

- Current requirement: The first\-generation system shall have simple functions and need to be associated with a language model\.

### \(II\) Database Team

- Core task: Store all project data and serve as the core hub for data interaction among all teams;

- System architecture: Take the V2 server as the communication core, with the available free server configuration of 4GB memory CPU (no GPU, the team needs to confirm its applicability);

- Interaction objects: Frontend (Mobile Team, Web Team), data providers (AI Team, Data Processing Team); not responsible for underlying algorithms and infrastructure;

- Document communication: Need to create a Documents or News; section to share information and synchronize with all responsible persons\.

### \(III\) Mobile Team

- Core scenario: For patients, the main functions include registration, login, sensor connection, etc\.;

- Specific requirements: Display patient use case diagrams and detailed descriptions, clarify that the alarm function is automatically operated by the service provider (not the responsibility of this team), and the data flow is that the V2 version sends payloads to the database;

- Schedule arrangement: Produce and test the simulation cover, conduct progress checks every Wednesday, and ensure the relevant work is completed within one week;

- Collaboration requirements: Discuss sensor collection and Bluetooth connection with the Sensor Team\.

### \(IV\) Web Team

- Core scenario: For doctors, covering three modules: account management, patient list management, and motion data display;

- Specific requirements:
        

    - Account management: Support registration (need to provide username, password, email, and doctor\&\#39;s license photo), and collaborate with the Database Team to define data specifications and interface documents;

    - Patient list management: Obtain patient information from the database;

    - Motion data display: Display the two knee joint angles collected by sensors and processed by data, and coordinate data conversion with the Sensor Team and Data Processing Team\.

- Additional responsibilities: Undertake administrator functions (only for the web terminal), support regular login/logout operations, and clarify interface specifications with the Database Team\.

### \(V\) Sensor Team

- Core areas: Wireless modules, communication ports, and reliable data transmission;

- Core tasks: The primary task is to provide sensors and knee joint angle data to the Data Processing Team; the secondary task is to discuss sensor collection and Bluetooth connection with the Mobile Team;

- Interaction requirements: Clarify interface specifications with the Data Processing Team to ensure efficient data transmission; coordinate sensor connection matters with the Mobile Team\.

### \(VI\) Data Processing Team

- Core work: Display data flow diagrams (Level 0 and Level 1 architectures) and clarify data sources and destinations;

- Data flow: Inputs are raw data packets from Group S1 and session requests from Group M1; outputs are standardized data (including knee joint angles, etc.) provided to Group V1;

- Collaboration requirements: Need to provide more detailed information on Group S2 and clarify interface specifications with all teams\.

## IV\. Project Time Requirements and Progress Management

- Deadline: April 30th, with only two weeks left for implementation\. Currently, focus on the development of the first\-generation system (the functions of the second and subsequent generations will be discussed later);

- Progress check: Conduct progress checks every Wednesday to ensure all teams advance as planned;

- Urgent actions: All teams need to privately discuss interface specifications immediately, improve and upload system design and interface specification documents\.

## V\. System Interaction and Collaboration Instructions

- Template recommendation process: The system automatically recommends rehabilitation templates, which are reviewed and supplemented by doctors;

- Patient use scenarios: All are the responsibility of the Mobile Team;

- Cross\-collaboration: The Sensor Team and the Data Processing Team work together to ensure efficient data transmission; all teams need to clarify interface specifications and strengthen communication\.

## VI\. Follow\-up Actions and Meeting Summary

- Document integration: The Logistics Group is responsible for integrating the documents of the six teams, completing and sharing them within the next two days;

- Closed\-door meetings: Arrange closed\-door meetings between teams to discuss details such as interface specifications in depth;

- Core requirements: All teams have clarified their division of labor and processes, need to strengthen communication, strictly abide by the deadline, and ensure the completion of the first\-generation system development before April 30th;

- Meeting conclusion: Clarify all tasks, encourage active communication between teams, and ensure the smooth progress of the project\.
